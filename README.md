## 問題與反思

**Q1. 你設計的哪一個 Skill 效果最好？為什麼？哪一個效果最差？你認為原因是什麼？**

> 我認為 `implement` 的 Skill 效果最好，因為它把前端、後端、資料庫與功能需求寫得很明確，AI 比較能一次產出接近可執行的結果。  
> 效果最差的是 `commit` Skill，因為它依賴實際環境中的 git 狀態、遠端設定與權限，這些通常不是單靠文字指令就能完全保證成功。

**Q2. 在用 AI 產生程式碼的過程中，你遇到什麼問題是 AI 沒辦法自己解決、需要你介入處理的？**

> 主要問題是環境設定與整合測試，例如 API key、套件版本、外部 API 回應格式，以及前後端欄位名稱是否一致。  
> AI 可以快速產生程式骨架，但在實際執行時，仍需要我手動修正錯誤、確認資料流與補上細節。
