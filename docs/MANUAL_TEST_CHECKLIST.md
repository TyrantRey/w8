# Manual Test Checklist

- [ ] **測試項目：首頁開啟**
  - 前置條件：伺服器已啟動
  - 操作步驟：
    1. 開啟瀏覽器
    2. 前往 `http://localhost:8000`
  - 預期結果：
    - 頁面可正常載入
    - 左側 session 區與右側聊天區可見

- [ ] **測試項目：建立新 session**
  - 前置條件：首頁已開啟
  - 操作步驟：
    1. 點擊「+ New Chat」
  - 預期結果：
    - 新 session 出現在左側列表
    - 自動切換到該 session

- [ ] **測試項目：切換歷史 session**
  - 前置條件：至少存在兩個 session
  - 操作步驟：
    1. 點擊左側不同 session
  - 預期結果：
    - 右側訊息區更新為對應聊天紀錄

- [ ] **測試項目：訊息顯示結構**
  - 前置條件：某個 session 中已有對話
  - 操作步驟：
    1. 查看訊息列表
  - 預期結果：
    - 每則訊息可看到 role
    - 每則訊息可看到 content
    - 每則訊息可看到 timestamp

- [ ] **測試項目：送出聊天訊息**
  - 前置條件：已選擇一個 session
  - 操作步驟：
    1. 在輸入框輸入文字
    2. 點擊 Send
  - 預期結果：
    - user 訊息成功顯示
    - assistant 回答成功顯示

- [ ] **測試項目：AI 回答**
  - 前置條件：已設定有效 GEMINI_API_KEY
  - 操作步驟：
    1. 送出一般提問
  - 預期結果：
    - 可收到 Gemini 回應內容

- [ ] **測試項目：上傳圖片或文件**
  - 前置條件：已選擇一個 session
  - 操作步驟：
    1. 選擇檔案
    2. 點擊 Upload
  - 預期結果：
    - 顯示上傳成功提示
    - 回傳 attachment_id

- [ ] **測試項目：附件引用**
  - 前置條件：已成功上傳檔案
  - 操作步驟：
    1. 輸入訊息
    2. 送出聊天
  - 預期結果：
    - 聊天請求可附帶 attachment_id
    - assistant 可參考檔案資訊回覆

- [ ] **測試項目：重新生成回答**
  - 前置條件：至少已有一則 assistant 訊息
  - 操作步驟：
    1. 點擊 assistant 訊息旁 regenerate
  - 預期結果：
    - 新增一則新的 assistant 訊息
    - 內容可能不同於原回答

- [ ] **測試項目：中止回應**
  - 前置條件：已選擇一個 session
  - 操作步驟：
    1. 在送出訊息後快速點擊 Stop
  - 預期結果：
    - 系統回傳 stopped 或提示已中止

- [ ] **測試項目：儲存使用者偏好**
  - 前置條件：首頁已開啟
  - 操作步驟：
    1. 輸入 name/language/tone
    2. 點擊 Save Preferences
  - 預期結果：
    - 顯示儲存成功
    - 重新整理後資料仍存在

- [ ] **測試項目：讀取使用者偏好**
  - 前置條件：已儲存偏好
  - 操作步驟：
    1. 重新整理頁面
  - 預期結果：
    - 表單顯示先前保存值

- [ ] **測試項目：外部工具 API**
  - 前置條件：網路可連線
  - 操作步驟：
    1. 輸入 `time: Asia/Taipei`
    2. 送出訊息
  - 預期結果：
    - assistant 回答中包含時區工具結果
    - 系統可正常使用 World Time API

- [ ] **測試項目：重整後資料持久化**
  - 前置條件：已有 session、訊息、偏好
  - 操作步驟：
    1. 重新整理頁面
  - 預期結果：
    - session 與訊息仍存在
    - 偏好仍可讀取

- [ ] **測試項目：空訊息錯誤處理**
  - 前置條件：已選擇一個 session
  - 操作步驟：
    1. 不輸入內容直接送出
  - 預期結果：
    - 系統顯示錯誤提示，不送出請求

- [ ] **測試項目：缺少 API key**
  - 前置條件：`.env` 未設定 GEMINI_API_KEY
  - 操作步驟：
    1. 送出聊天訊息
  - 預期結果：
    - 系統回傳 API key 缺失的錯誤訊息

- [ ] **測試項目：無效 session**
  - 前置條件：使用不存在的 session_id 呼叫 API
  - 操作步驟：
    1. 手動測試錯誤 API
  - 預期結果：
    - 後端回傳 404
